# Project Structure

This document explains the organization of the Publish to Dev.to Action repository.

## Directory Layout

```
hugo-to-devto-action/
├── .github/
│   └── workflows/
│       ├── build.yml           # Build and validate action
│       ├── test.yml            # Test action with sample post
│       ├── release.yml         # Create releases on tags
│       └── example-usage.yml   # Example workflow for users
├── docs/
│   ├── EXAMPLES.md            # Usage examples and patterns
│   └── PROJECT_STRUCTURE.md   # This file
├── scripts/
│   └── setup.sh               # Development setup script
├── src/
│   └── index.ts               # Main action source code
├── test/
│   └── sample-post.md         # Sample markdown for testing
├── dist/                      # Compiled action (generated)
│   └── index.js               # Bundled action code
├── action.yml                 # Action metadata and interface
├── package.json               # Node.js dependencies
├── tsconfig.json              # TypeScript configuration
├── .gitignore                 # Git ignore rules
├── LICENSE                    # MIT License
├── README.md                  # Main documentation
├── SETUP.md                   # Setup guide for users
├── CONTRIBUTING.md            # Contribution guidelines
├── CHANGELOG.md               # Version history
├── scripts/
│   ├── setup.sh               # Development setup
│   └── publish-to-devto.ts    # Original standalone script
```

## Key Files

### Action Definition

- **action.yml**: Defines the action's inputs, outputs, and runtime configuration
  - Inputs: api-key, file-path, base-url
  - Outputs: article-url, article-id
  - Runtime: Node.js 20

### Source Code

- **src/index.ts**: Main action logic
  - Reads and parses markdown files
  - Extracts YAML frontmatter
  - Publishes to Dev.to API
  - Uses @actions/core for GitHub Actions integration

### Build Output

- **dist/index.js**: Compiled and bundled action code
  - Generated by @vercel/ncc
  - Includes all dependencies
  - Must be committed to repository
  - Used by GitHub Actions runtime

### Configuration

- **package.json**: Node.js project configuration
  - Dependencies: @actions/core
  - Dev dependencies: TypeScript, @vercel/ncc
  - Build script: `ncc build src/index.ts`

- **tsconfig.json**: TypeScript compiler options
  - Target: ES2022
  - Module: CommonJS
  - Strict mode enabled

### Documentation

- **README.md**: Main documentation with usage examples
- **SETUP.md**: Step-by-step setup guide for new users
- **CONTRIBUTING.md**: Guidelines for contributors
- **CHANGELOG.md**: Version history and release notes
- **docs/EXAMPLES.md**: Comprehensive usage examples
- **docs/PROJECT_STRUCTURE.md**: This file

### Testing

- **test/sample-post.md**: Sample markdown file for testing
- **.github/workflows/test.yml**: Automated testing workflow
- **.github/workflows/build.yml**: Build validation workflow

### Scripts

- **scripts/setup.sh**: Development environment setup
- **scripts/publish-to-devto.ts**: Original standalone script (for local testing)

## Development Workflow

### 1. Making Changes

```bash
# Edit source code
vim src/index.ts

# Build the action
npm run build

# Test locally (optional)
export DEVTO_API_KEY="your-key"
npx tsx scripts/publish-to-devto.ts test/sample-post.md
```

### 2. Building

The build process:
1. TypeScript compiles `src/index.ts`
2. @vercel/ncc bundles code and dependencies
3. Output written to `dist/index.js`
4. Source maps and licenses generated

### 3. Testing

```bash
# Run build validation
npm run build

# Test with GitHub Actions (push to test branch)
git push origin feature-branch

# Manual test via workflow_dispatch
# Go to Actions tab → Test Action → Run workflow
```

### 4. Releasing

```bash
# Update version in package.json
npm version patch  # or minor, major

# Update CHANGELOG.md
vim CHANGELOG.md

# Commit changes
git add .
git commit -m "Release v1.0.1"

# Create and push tag
git tag v1.0.1
git push origin main --tags

# GitHub Actions will create release automatically
```

## Important Notes

### The dist/ Directory

⚠️ **Critical**: The `dist/` directory must be committed to the repository!

GitHub Actions cannot run build steps when using an action, so the compiled code must be available in the repository. This is different from typical Node.js projects where build artifacts are gitignored.

### Version Management

- Use semantic versioning (MAJOR.MINOR.PATCH)
- Update CHANGELOG.md for each release
- Create git tags for releases (v1.0.0, v1.1.0, etc.)
- GitHub Actions handles release creation automatically

### Dependencies

- Keep dependencies minimal
- Only @actions/core is required at runtime
- Dev dependencies are not included in dist/
- Update dependencies regularly for security

## Adding New Features

1. Update `src/index.ts` with new functionality
2. Add new inputs to `action.yml` if needed
3. Update `README.md` with usage examples
4. Add examples to `docs/EXAMPLES.md`
5. Update `CHANGELOG.md`
6. Build and test
7. Commit all changes including `dist/`
8. Create PR or release
